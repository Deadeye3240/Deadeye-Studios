## Deadeye Studio v1.0.1

Bug-fix release for a startup hang introduced in v1.0.0.

### Fixed: app stuck on "Starting interface..."

**What happened (v1.0.0):** On the second launch (and every launch after), if you had previously closed Deadeye Studio with a project folder open, the app tried to show a **"Restore Session?"** dialog while the full-screen loading overlay was still visible. The dialog rendered **behind** the loading screen, so the app appeared frozen on **"Starting interface..."** with no way to continue.

**Who was affected:** Any user on v1.0.0 who closed the app with a workspace open, then launched it again. First-time launches were unaffected.

**What we changed:**
- Dismiss the loading screen before session-restore prompts run.
- Raise the dialog overlay above the loading layer so modals are always visible.
- Skip session restore automatically when the saved workspace folder no longer exists.
